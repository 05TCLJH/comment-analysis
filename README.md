# 美伊战争多源网络评论分析

这个项目当前处于骨架阶段，目标是先把“采集 -> 清洗 -> 存储 -> 分析 -> 展示”这条链路的目录和入口整理清楚，后续再逐步补充具体实现。

## 当前骨架思路

- `src/comment_analysis/` 是项目主包，后续所有核心代码都放这里。
- `entry/` 放项目入口，负责启动不同阶段的流程。
- `crawlers/` 放各个平台的爬虫适配器。
- `parsers/` 放网页内容解析逻辑。
- `pipeline/` 放流程编排逻辑。
- `storage/` 放数据落地逻辑。
- `analysis/` 放情感、关键词、主题等分析逻辑。
- `visualization/` 放图表和结果展示逻辑。
- `utils/` 放通用工具。

## 现阶段建议

1. 先确认第一个爬取来源。
2. 先定义统一的数据字段。
3. 先把“采集入口”和“分析入口”分开。
4. 先让目录稳定，再逐步补代码。
